# Project Conventions

## TDGS — Hard Rules

1. **Деньги/бумаги:** Правила в [tdgs-sirius-team-rules.md](./tdgs-sirius-team-rules.md). При ошибках API — НЕ придумывать решение, спросить человека.
2. **OTC seller rule:** seller и buyer НЕ могут иметь один ank_no — всегда отдельный клиент как контрагент.
3. **Trade numbers:** API НЕ возвращает реальный trd_no — всегда спрашивать у пользователя.
4. **Batch:** При >2 однотипных операций — ВСЕГДА один bash-скрипт (один логин → массив → foreach). Шаблоны: `.kilo/skills/tdgs/templates/`.
5. **Лог-файл:** При любой TDGS-задаче — всегда создавать лог в `tdgs-work/` (шаблон: `tdgs-work/test-clients.md`).
6. **Конфлюенс/веб:** Только текст, НЕ скачивать картинки. Картинки = 50-200K токенов каждая.
7. **Пошаговое чтение:** Читать только то, что нужно на текущем шаге. SKILL.md — точечно по секциям, шаблоны — только при генерации скрипта.

## Эндпоинты, инструменты, environment — в SKILL.md

Активируй навык `tdgs` для деталей. AGENTS.md содержит только критические правила.
