# TDGS: Правила налоговой команды (Sirius-team)

## Деньги и бумаги

### НИКОГДА не заводить клиентам без запроса

При создании сделок (биржевых или OTC) **НЕ заводить** деньги или бумаги на счета клиентов самостоятельно.

External/Internal Transfers — **только** по явному запросу в задаче.

### OTC контрагент — отдельный клиент

При OTC-сделках:
- Seller и buyer — **разные** клиенты (разные ank_no)
- Ничего не заводить на счёт контрагента
- Контрагент создаётся как обычный клиент (1 treaty, QUAL)

## Batch-скрипты

### При >2 однотипных операций — batch

Один логин → массив данных → foreach → вывод.

**Шаблоны в `templates/`:**

| Операция | Файл |
|----------|------|
| Биржевые сделки | `ps-batch-exchange.ps1` |
| OTC сделки | `ps-batch-otc.ps1` |
| Создание клиентов | `ps-batch-create-users.ps1` |
| Внешние зачисления | `ps-batch-transfer.ps1` |

**Типовая последовательность сложной задачи:**
```
1: Логин + create/users → собрать acc_code
2: Логин + create/user/ttk (если нужна)
3: Логин + foreach: ВСЕ биржевые сделки
4: Логин + foreach: ВСЕ OTC сделки
5: write → лог-файл
```

~5-7 bash-вызовов вместо 50+.

## Корпоративные действия (КД)

Требуют помимо API ещё ручную работу в БД. Детали: [db-operations-advanced.md](./db-operations-advanced.md).

При запросе: выполнить API-часть, затем **сообщить пользователю** список того, что нужно докрутить в БД.

### Конвертация

**Что делает агент (через API):**
1. Изначальная покупка бумаги X
2. Списание бумаги X + зачисление бумаги Y — количество в трансферах совпадает

**Что пользователь докручивает в БД самостоятельно:**
- `tax_data.tax.substitution` — заполнить
- В трансферах: одинаковый `group_no`
- В трансферах: `income_type = 61`
- В трансферах: `as_p_code = p_code` парного трансфера

Таблицы-источники: `dbo.a_all_transfers` или `dbo.extra_transfers`

### Сплит / Консолидация

**Что делает агент (через API):**
1. Изначальная покупка бумаги X
2. Списание бумаги X + зачисление бумаги Y

**Что пользователь докручивает в БД самостоятельно:**
- В трансферах: `income_type = 65`
- Заполнить таблицу `tax_calc.tax.SplitOperation`: `split_type = 1`
