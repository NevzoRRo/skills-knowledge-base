# Changelog

## [2026-07-17] — Режим редактирования протокола + corrections.json

### Добавлено
- **3-й режим "Редактирование"** в скилл `meeting-protocol` (наряду со стандартным и расширенным). При выборе запрашивает ссылку на существующий протокол в Confluence, загружает содержимое через API и позволяет вносить изменения.
- **Файл `corrections.json`** — хранит пользовательские корректировки (опечатки в ФИО, названиях встреч, датах) для их автоматического применения при последующих вызовах скилла.
- **Автокорректировки** — перед формированием протокола любого типа скилл применяет сохранённые исправления из `corrections.json`.
- **Механизм обновления SKILL.md** — при содержательных изменениях (добавление задач, решений, участников) скилл спрашивает, нужно ли запомнить правило, и при согласии обновляет SKILL.md.
- **Классификация изменений** — 9 типов (`typo_name`, `typo_title`, `typo_date`, `add_task`, `modify_task`, `add_decision`, `add_participant`, `content`, `other`) с разной логикой обработки.

### Изменено
- **Описание скилла** в YAML frontmatter — теперь включает упоминание редактирования и запоминания корректировок.
- **Блок "ВАЖНО"** — добавлено упоминание 3-го режима и строгое правило ожидания ответа перед следующим вопросом.
- **Шаг 2** — теперь предлагает 3 варианта (standard / extended / edit) вместо 2.
- **Шаг 1** (сохранение) — пропускается для режима "Редактирование".
- **Шаг 3** — перед извлечением данных применяются корректировки из `corrections.json`.
- **Итоговый флоу** — перестроен: Часть 1 → [Часть 3, если edit] → Часть 2 (письма).
- **Примеры** — добавлен пример для режима редактирования.

### Удалено
- **Cимлинк `skills-knowledge-base`** — заменён на прямую директорию `meeting-protocol/`.

### Инфраструктурное
- Добавлен `CHANGELOG.md` в корень проекта.
