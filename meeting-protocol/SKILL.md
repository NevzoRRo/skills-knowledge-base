## Файл corrections.json

**Расположение:** рядом с SKILL.md (`~/.config/opencode/skills/meeting-protocol/corrections.json`)

Этот файл хранит пользовательские корректировки, чтобы при следующих вызовах скилла данные автоматически подставлялись исправленными.

**Структура:**
```json
{
  "name_corrections": {
    "Иванов Иван": "Иванов И.И."
  },
  "title_corrections": {
    "Встреча по проекту": "Еженедельный статус по проекту Альфа-Инвестиции"
  },
  "date_corrections": {
  },
  "task_corrections": {
  },
  "decision_corrections": {
  },
  "custom_corrections": {}
}
```

**Где используются:**
- `name_corrections` — при чтении участников и ответственных из транскрипции
- `title_corrections` — при формировании заголовка протокола
- `date_corrections` — при указании даты встречи
- `task_corrections` — при формулировке задач
- `decision_corrections` — при формулировке решений

**Порядок применения:**
1. Прочитай транскрипцию
2. Перед формированием протокола (любого типа) — **примени все корректировки** из `corrections.json`
3. Если фамилия в транскрипции совпадает с ключом в `name_corrections` — используй значение
4. Если название встречи совпадает с ключом в `title_corrections` — используй значение

## Часть 1: Создание протокола в Confluence

```text
2. URL страницы (скопируй из браузера)
```

Если выбран режим "Редактирование" — этот шаг пропускается.

### Шаг 2: Спросить режим работы / тип протокола

Перед анализом транскрипции ОБЯЗАТЕЛЬНО спроси пользователя, какой режим нужен:

```text
Выберите режим работы:
1. Стандартный протокол — краткий структурированный протокол
2. Расширенный протокол — управленческий протокол с детализацией
3. Редактирование — исправить существующий протокол в Confluence
```

Если доступен `question` tool, используй его с тремя вариантами выбора.

**Правила:**
- Если выбран **Стандартный протокол** — извлекай только базовые сущности и не задавай дополнительные уточняющие вопросы, кроме критически необходимых; `protocol_type = "standard"`
- Если выбран **Расширенный протокол** — обязательно задай дополнительные уточняющие вопросы, если в транскрипции не хватает данных для качественного управленческого протокола; `protocol_type = "extended"`
- Если выбран **Редактирование** — `protocol_type = "edit"`. **НЕМЕДЛЕННО перейди к Части 3**. Не спрашивай куда сохранять, не читай транскрипцию, не формируй HTML заново.

### Шаг 3: Прочитать транскрипцию и применить корректировки

**Перед формированием протокола — примени корректировки из `corrections.json`:**

Прочитай файл `~/.config/opencode/skills/meeting-protocol/corrections.json`:
- `name_corrections` — исправь фамилии/ФИО участников и ответственных
- `title_corrections` — исправь название встречи
- `date_corrections` — исправь дату
- `task_corrections` — скорректируй формулировки задач
- `decision_corrections` — скорректируй формулировки решений

```bash
uv run --with requests python -c "
import json, pathlib
path = pathlib.Path.home() / '.config' / 'opencode' / 'skills' / 'meeting-protocol' / 'corrections.json'
if path.exists():
    corr = json.loads(path.read_text(encoding='utf-8'))
    print(json.dumps(corr, ensure_ascii=False, indent=2))
else:
    print('{}')
"
```

**Правила применения:**
1. Если фамилия из транскрипции совпадает с ключом в `name_corrections` — используй значение
2. Если название встречи совпадает с ключом в `title_corrections` — используй значение
3. Аналогично для дат, задач, решений

После применения корректировок извлеки базовые данные:

- Дату встречи
- Название встречи
- Участников (список ФИО)
- Ключевые темы
- Принятые решения
- Задачи с ответственными и дедлайнами

#### Если выбран "Стандартный протокол"

Сформируй краткий и структурированный протокол без лишних уточнений.

Включи:
- Дату
- Название встречи
- Участников
- Цель встречи (кратко)
- Повестку
- Принятые решения
- Задачи с ответственными и дедлайнами
- Краткое резюме

Задавай уточняющие вопросы только если без них нельзя корректно сформировать протокол:
- неясна дата
- непонятно, кто ответственный
- задача совсем неконкретна
- невозможно определить, было ли принято решение

#### Если выбран "Расширенный протокол"

Сформируй управленческий протокол повышенной детализации.

Обязательно дополнительно извлеки или уточни:

1. **Контекст / Предыстория**
   - откуда взялась встреча
   - что было до (предыдущие встречи, решения, документы)
   - причина созыва

2. **Кворум**
   - кто присутствовал (фактически)
   - кто должен был быть (по плану)
   - кто отсутствовал и по какой причине (если известно)

3. **Детальную цель встречи**
- бизнес-контекст
- проблема или возможность
- ожидаемый результат встречи
- какой уровень решения ожидался: информирование / выработка предложений / принятие решения / эскалация

2. **Статус обсуждения по вопросам**
- решено
- требует доработки
- отложено
- вынесено на эскалацию

3. **Подробные action points**
Для каждой задачи зафиксируй:
- что именно нужно сделать
- какой ожидается результат или артефакт
- кто персонально отвечает
- кто соисполнители, если есть
- срок
- критерий готовности

4. **Спорные и открытые вопросы**
Для каждого такого пункта выдели:
- суть спорного вопроса
- позиции сторон
- что мешает договориться
- следующий шаг
- нужна ли эскалация

5. **Риски и зависимости**
Если они явно звучали во встрече, извлеки:
- риск
- влияние
- зависимость от другой команды, решения или срока
- что нужно сделать для снятия риска

6. **Ожидаемые follow-up действия**
- что нужно проверить после встречи
- какие материалы должны быть досланы
- какие следующие встречи или контрольные точки нужны

#### Правила формулировки цели встречи для расширенного протокола

Цель встречи должна быть максимально конкретной и не должна ограничиваться фразами вроде:
- "обсудить статус"
- "синхронизироваться"
- "рассмотреть вопрос"

Формулируй цель в структуре:
- бизнес-контекст
- предмет обсуждения
- ожидаемый итог встречи
- горизонт решения

Пример хорошей формулировки:

```text
Цель встречи: согласовать целевой вариант запуска MVP витрины X, определить зоны ответственности между командами Less, аналитики и бизнеса, а также зафиксировать перечень действий и сроков для выхода на подготовку релиза до 01.09.2026.
```

Если цель встречи в транскрипции размыта, задай уточняющие вопросы:

```text
Для расширенного протокола нужно уточнить цель встречи:
1. Какую основную бизнес-проблему или задачу вы хотели решить?
2. Какой результат считался успешным по итогам встречи?
3. Вы хотели принять решение, подготовить предложения, синхронизировать позиции или эскалировать вопрос?
```

#### Правила формулировки action points

Каждая задача должна быть сформулирована так, чтобы из неё было понятно:
- кто делает
- что делает
- какой должен быть результат
- к какому сроку
- как понять, что задача выполнена

Плохо:
- "разобраться"
- "обсудить"
- "подготовить вариант"

Хорошо:
- "Иванов И.И. готовит и направляет участникам встречи сводный список интеграционных ограничений по системе X до 25.07.2026; результатом считается документ в Confluence с подтверждением от архитектуры."

Если задача в транскрипции неконкретна, задай уточняющий вопрос по недостающим полям.

#### Правила уточняющих вопросов для расширенного протокола

Если данных недостаточно, ОБЯЗАТЕЛЬНО задай уточняющие вопросы пользователю.

Задавай вопросы только по недостающим данным, а не всем списком сразу.

Приоритет уточнений:
1. цель встречи
2. спорные вопросы
3. action points
4. сроки и ответственные
5. риски и зависимости

Пример:

```text
Для расширенного протокола не хватает нескольких уточнений:

1. Цель встречи:
- какую основную проблему вы хотели решить?
- какой результат считался успешным по итогам встречи?

2. По задаче "подготовить вариант":
- кто основной ответственный?
- какой должен быть результат: документ, согласование, расчёт, презентация?
- какой срок?

3. По спорному вопросу о приоритетах:
- в чём различались позиции сторон?
- нужен ли вынос на эскалацию?
```

### Шаг 4: Сформировать HTML протокола

Используй шаблон в зависимости от значения `protocol_type`:
- `standard` -> стандартный HTML-шаблон
- `extended` -> расширенный HTML-шаблон

Перед генерацией HTML проверь:

- цель встречи заполнена осмысленно
- в задачах нет безличных формулировок
- у каждой задачи есть ответственный
- спорные вопросы не смешаны с принятыми решениями

### Шаг 5: Создать страницу в Confluence

Используй Python + requests:



## Часть 3: Редактирование существующего протокола

### Общее описание режима

Режим **Редактирование** предназначен для исправления уже существующего протокола в Confluence, а также для запоминания корректировок пользователя для будущего использования.

**Когда вызывается:** пользователь выбрал режим 3 ("Редактирование") на Шаге 2.

**Что делает:**
1. Запрашивает ссылку на существующий протокол
2. Получает текущее содержимое страницы через Confluence API
3. Спрашивает, что нужно изменить
4. Применяет изменения в Confluence
5. Если изменения касаются опечаток в ФИО, названиях — сохраняет корректировку в `corrections.json`
6. Если изменения касаются содержательной части — спрашивает, обновить ли скилл
7. При согласии — модифицирует `SKILL.md` соответствующим образом

Если пользователь дал не URL, а ID страницы — используй ID напрямую.

### Шаг 3.2: Получить текущее содержимое

Используй Confluence API чтобы получить содержимое страницы:

```bash
uv run --with requests python -c "
import requests, json
token = 'TOKEN'

url = f'https://confluence.moscow.alfaintra.net/rest/api/content/{page_id}?expand=body.storage,version'
r = requests.get(url, headers={'Authorization': f'Bearer {token}'}, verify=False)
data = r.json()
print('Title:', data.get('title'))
print('Version:', data['version']['number'])
print('Body:', data['body']['storage']['value'])
"
```

### Шаг 3.5: Сохранить корректировки в corrections.json

**Для исправлений опечаток (typo_name, typo_title, typo_date):**

Сразу (без дополнительных вопросов) добавь запись в `corrections.json`:

```bash
uv run --with requests python -c "
import json, pathlib
path = pathlib.Path.home() / '.config' / 'opencode' / 'skills' / 'meeting-protocol' / 'corrections.json'
corr = json.loads(path.read_text(encoding='utf-8'))
corr['name_corrections']['НеправильноеИмя'] = 'ПравильноеИмя'
# или для title_corrections:
corr['title_corrections']['НеправильноеНазвание'] = 'Правильное название'
path.write_text(json.dumps(corr, ensure_ascii=False, indent=2), encoding='utf-8')
print('OK')
"
```

**ВАЖНЫЕ ПРАВИЛА:**
1. Файл `corrections.json` лежит в двух местах: `~/.config/opencode/skills/meeting-protocol/corrections.json` (рабочий, используется при вызове скилла) и `meeting-protocol/corrections.json` в репозитории (для git).
2. **При любом изменении corrections.json обновляй ОБА файла:**
   - `~/.config/opencode/skills/meeting-protocol/corrections.json` (чтобы скилл сразу подхватил)
   - `meeting-protocol/corrections.json` в репозитории (чтобы изменения не потерялись при пуше)

### Шаг 3.6: Обновить страницу в Confluence

Сформируй новый HTML на основе текущего содержимого с внесёнными изменениями.

Используй API для обновления страницы (PUT с version number):

```bash
uv run --with requests python -c "
import requests, json
token = 'TOKEN'
page_id = 'PAGE_ID'
url = f'https://confluence.moscow.alfaintra.net/rest/api/content/{page_id}'
version_num = CURRENT_VERSION + 1
page_data = {
    'id': page_id,
    'type': 'page',
    'title': 'НОВЫЙ ТАЙТЛ',
    'version': {'number': version_num},
    'body': {'storage': {'value': 'НОВЫЙ_HTML', 'representation': 'storage'}}
}
r = requests.put(url, headers={'Authorization': f'Bearer {token}', 'Content-Type': 'application/json'}, data=json.dumps(page_data), verify=False)
print(r.json().get('id'))
"
```

3. Обнови оба файла:
   - `~/.config/opencode/skills/meeting-protocol/SKILL.md`
   - `meeting-protocol/SKILL.md` в репозитории

**Если пользователь ответил "нет" или что-то неопределённое:**

### Поиск email в Outlook

**Используй Python с pypiwin32:**

```bash
uv run --with pypiwin32 python -c "
import win32com.client
import pythoncom

pythoncom.CoInitialize()
outlook = win32com.client.Dispatch('Outlook.Application')
ns = outlook.GetNamespace('MAPI')
al = ns.AddressLists.Item('All Users')

users = []
for e in al.AddressEntries:
    try:
        u = e.GetExchangeUser()
        if u:
            users.append({'name': u.Name, 'email': u.PrimarySmtpAddress})
    except:
        pass

found = [u for u in users if 'ФАМИЛИЯ' in u['name']]
for u in found:
    print(u['name'], '=>', u['email'])
"
```
